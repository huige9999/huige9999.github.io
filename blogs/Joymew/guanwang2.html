    <template>
  <svg :class="svgClass" aria-hidden="true">
    <use :xlink:href="iconName" />
  </svg>
</template>

<script>
export default {
  name: 'svgIcon',
  props: {
    iconClass: {
      type: String,
      required: true,
    },
    className: {
      type: String,
      default: '',
    },
  },
  computed: {
    svgClass() {
      if (this.className) {
        return `svg-icon ${this.className}`;
      }
      return 'svg-icon';
    },
    iconName() {
      return `#icon-${this.iconClass}`;
    },
  },
};
</script>

<style scoped>
.svg-icon {
  width: 1em;
  height: 1em;
  fill: currentColor;
  stroke: currentColor;
  overflow: hidden;
}
</style>

    # 归纳总结

    1. 关于require和require.context的区别

    • require和require.context都是用来引入模块的
    • require是没有范围限制的，即：你不知道它会引入哪里的模块
    • require.context是有范围限制的，即：你能够确定它所引入的模块是哪个范围下的？哪些模块?

    require.context()只是得到一个有范围限制的require，后续你还得调用它引入模块; requireContext.keys()能获取到该范围下的所有模块路径; map方法用来遍历每一个路径，将路径作为参数调用requireContext()引入模块。

    const requireAll = (requireContext) => requireContext.keys().map(requireContext);

    2. 关于chainWebpack的链式调用怎么理解?

    我们来回顾下webpack的基本配置

    module.exports = {
  mode: 'development', // 设置模式为开发模式
  entry: './src/index.js', // 入口文件
  output: {
    filename: 'bundle.js', // 输出文件名称
    path: path.resolve(__dirname, 'dist'), // 输出文件路径
  },
  module: {
    rules: [
      {
        test: /\.js$/, // 匹配所有 .js 文件
        exclude: /node_modules/, // 排除 node_modules 文件夹
        use: {
          loader: 'babel-loader', // 使用 Babel 转译
          options: {
            presets: ['@babel/preset-env'],
          },
        },
      },
      {
        test: /\.(png|jpe?g|gif|svg)$/, // 匹配图片文件
        use: [
          {
            loader: 'file-loader', // 使用 file-loader
            options: {
              name: '[name].[hash].[ext]', // 自定义输出文件名称
              outputPath: 'images/', // 输出到 images 文件夹
            },
          },
        ],
      },
    ],
  },
};

    • config对应CommonJs导出的对象

    • .entry()、.out()对应着配置对象的每个属性，即：会返回属性对应的对象 所以也就意味着每次链式调用返回的对象可能会一层一层往下(因为配置对象值可能会嵌套多层)，config对象是起点

    • end()方法可以返回上一层,比如 .entry() 会来到entry对象这一层，调用end()又会返回config对象这一层
