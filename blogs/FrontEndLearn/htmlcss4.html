    # BFC(块级格式化上下文)

    # 概念详解

    它是一块独立的渲染区域，规定了该区域中常规流块盒的布局

    解析： 常规流块盒的布局：

    • 水平方向上撑满包含块
    • 垂直方向上依次摆放
    • 外边距无缝相邻会合并
    • 它的自动高度和摆放位置无视浮动元素

    BFC渲染区域： 由某个HTML元素创建，以下元素会在内部创建BFC区域:

    • 根元素 意味着html元素创建的BFC区域覆盖了网页中的所有元素
    • 浮动和绝对定位元素
    • oveflow不等于visible的块盒 意味着设置了oveflow属性

    独立的： 不同的BFC区域渲染时互不干扰

    具体规则： 创建BFC的元素，

    • 它的自动高度需要计算浮动元素
    • 它的边框盒不会与浮动元素重叠 不设置会无视浮动元素，设置了会避开浮动元素 避开的原理：自动设置了margin 适用场景：左盒子固定、有盒子宽度自适应剩余空间
    • 不会和它的子元素进行外边距合并

    # 小结归纳

    • 创建独立的渲染区域
    • 解决外边距合并、浮动带来的高度塌陷问题
    • 创建BFC的副作用最小的方式：overflow: hidden、auto、scroll

    # 布局

    # 多栏布局

    • 两栏布局 左元素设置float:left，右元素常规流块盒创建BFC(宽度会撑满剩余空间)

    • 三栏布局 左右分别:float:left、right 中间常规流创建BFC (创建BFC的元素独立渲染会避开浮动盒子)

    # 等高布局

    通常情况：左侧是菜单高度会不够，右侧内容高度会超出左侧高度，所以要解决侧边高度不够的问题。

    解决方式：

    • css3的弹性盒布局(弹性item高度默认拉伸)

    • js控制高度

    • 伪等高 height设置非常大，比如:height: 10000px; 然后设置margin-bottom: -9990px; 那么盒子的总高度只有10px

      原理：我们希望父元素高度由右侧元素决定，所以设置margin-bottom: -9990px让左侧盒子整体的高度很小，最后再给父元素加上overflow: hidden; 设置height:10000px的目的是弥补左侧盒子的高度不够高的问题

      记忆：给左侧盒子足够高度同时让父元素高度取决于右侧盒子的高度

    # 后台布局

    header、sidebar、content

    • 整个容器固定定位宽高相对视口100%
    • header定高，绝对定位在顶部
    • 内容区域也是100%宽高，设置padding-top: header高度
    • sidebar定宽，高度100%，左浮动,溢出滚动
    • content设置BFC避开sidebar，高度100%

    # 行高的取值

    1. 如果是px，则按px为准 因为px是绝对单位 适用情况：多行文本字体大小相同，可以在父元素用px单位统一设置行高 但如果每一行的字体大小不一样，父元素就不能用px单位统一设置行高了

    2. 如果直接写数字 先继承给子元素然后计算出px，所以子元素会按照得到各自字体大小倍数的行高，所以这种方式是能在父元素统一设置行高的!

    3. 如果是em单位 em表示当前元素的字体大小，比如2em是当前的2倍

    如果在父元素设置2em，它会先计算出2em=多少px，然后再继承给子元素，所以这个单位也不能统一设置行高

    1. 如果是百分比 和em一样，也是相对当前元素的字体大小，所以也不能统一设置行高

    理解记忆：写数值会直接继承给子元素，写em和百分比都是相对单位要先计算出px值才能继承给子元素

    # body的背景

    # 关于背景颜色

    引入一个概念：画布区域

    画布区域的背景：

    • 最小宽度为视口宽度
    • 最小高度为视口高度
    • 画布的实际宽高会随着实际网页内容的变化而变化(有滚动条的情况)

    HTML元素的背景颜色： 覆盖画布

    给body设置背景颜色： 如果HTML元素有背景颜色，则一切正常(覆盖边框盒) 如果HTML元素没有背景颜色，则body背景来覆盖画布

    理解记忆：

    • 画布不是指html5中的canvas，就是一块区域承载着网页内容！
    • 设置这个规则是为了方便你给整个网页设置背景!

    # 关于背景图

    1. 背景图宽度的百分比相对于视口
    2. 背景图高度的百分比相对于网页高度
    3. 背景图的横向位置百分比相对于视口
    4. 背景图的纵向位置百分比相对于网页高度

    # 小结归纳：

    给body元素设置背景，先看HTML元素有没有背景，如果没有就注意参照物是视口还是网页

    # 堆叠上下文

    堆叠上下文（stack context），它是一块区域，这块区域由某个元素创建(类似于BFC)，它规定了该区域中的内容在Z轴上排列的先后顺序。

    # 创建堆叠上下文的元素

    1. html元素（根元素）
    2. 设置了z-index（非auto）数值的定位元素

    # 同一个堆叠上下文中元素在Z轴上的排列

    从后到前的排列顺序：

    1. 创建堆叠上下文的元素的背景和边框
    2. 堆叠级别(z-index, stack level)为负值的堆叠上下文
    3. 常规流非定位的块盒
    4. 非定位的浮动盒子
    5. 常规流非定位行盒
    6. 任何 z-index 是 auto 的定位子元素，以及 z-index 是 0 的堆叠上下文
    7. 堆叠级别为正值的堆叠上下文

    每个堆叠上下文，独立于其他堆叠上下文，它们之间不能相互穿插。

    # 小结归纳：

    比较两个元素的层级关系： 1. 先判断两个元素是否在同一个堆叠上下文中， 2. 如果在，按照如下规则判断： 从低到高的顺序： 1) 创建堆叠上下文元素的背景和边框 2) z-index为负数的元素 3) 常规流块盒->浮动盒子->常规流行盒 4) 定位元素或者z-index为0的元素 5) z-index为正数的元素 3. 如果不在，分别找出它们所属的上下文再判断是不是在同一个堆叠上下文中

    补充：谁创建了堆叠上下文，就把它和它的后代元素当成一个整体

    # svg

    # 基本概念

    svg: scalable vector graphics，可缩放的矢量图(区别于像素点组成的位图)

    1. 该图片使用代码书写而成
    2. 缩放不会失真
    3. 内容轻量(仅仅是指令数据而不是若干像素点信息)

    原理： 普通图片是一个个像素点组成的 svg图片定义好指令(比如：从这个坐标到另外一个坐标连成一条直线或弧线)，浏览器根据指令画出图形

    局限性： 它只能实现简单的图片，现实生活中高清相机拍出来的图片svg很难描述出来!

    # 使用方式

    由于svg本质是指令代码所以使用方式：

    1. 嵌入网页
    2. 单独作为一个文件

    svg代码：用xml格式书写

    # 代码分析

     <!-- 1.svg描述一张图片，width和height属性设置图片的原始大小，也可以用style属性设置背景颜色。 -->
<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100">
  <!-- 画一个矩形,左上角坐标:(5,5)、宽高：80,90,填充色:red,描边颜色:#000,描边宽度:5 -->
        <rect x="5" y="5" width="80" height="80" fill="red" stroke="#000" stroke-width="5" />
    <!-- 画一个圆形,中心点坐标:(30,30)、半径: 30,填充色: skyblue -->
         <circle cx="200" cy="200" r="30" fill="skyblue" />
      <!-- 画一个椭圆，长轴半径：100，短轴半径：50，中心点坐标：(200,200)，填充色：透明，描边颜色：#000，描边宽度：5 -->
        <ellipse cx="200" cy="200" rx="100" ry="50" fill-opacity="0" stroke="#000" stroke-width="5" />
          <!-- 画一个线条，起点坐标：(100,100)，终点坐标：(300,300)，颜色：#000，宽度：5 -->
        <line x1="100" y1="100" x2="300" y2="300" stroke="#000" stroke-width="5" />
        <!-- 画一条折线，点坐标：(300,100)、(350,100)、(350,150)、(400,400)，颜色：#000，宽度：5 -->
         <!-- 如果fill不设置为none,它会自动帮你连成封闭区域并填充颜色。 -->
        <polyline points="300,100 350,100 350,150" stroke="#000" stroke-width="5" fill="none" />
      <!-- 通用：路径写法(指令写法) -->
         <!-- d表示指令 -->
         <!-- M0 0表示将画笔移动到坐标(0,0) -->
          <!-- L300 300表示从当前位置画一条直线到坐标(300,300) -->
         <path d="M0 0 L300 300" stroke="#000" stroke-width="5" />
           <!-- 描边等样式也可以直接写到style属性中 -->
         <path d="M0 0 L300 300" stroke="#000" style="stroke: #000;stroke-width: 5;" />

</svg>

    # 数据链接(dataurl)

    # 概念引入

    <img src='./img/test.png'>
    1

    分析：将src地址指向的文件读出来并解析成计算机能理解的数据，再把数据当成图片显示出来!

    而dataurl是将文件的二进制数据直接写在src属性中！(二进制是无法直接写出来的，所以需要base64编码。) 注：我们写出来的0101是个可打印的字符串而不是真正的二进制数据

    # 语法

    data:MIME,base64编码数据

     <!-- 写文件地址 -->
 <link rel="stylesheet" href="./1.css">
 <!-- 用dataurl直接写上css文件内容 -->
 <link rel="stylesheet" href="data:text/css,h1{color: red}">

    # 意义

    优点：减少HTTP请求 缺点：文件体积大，不利于缓存

    # 适用场景

    适用于小图片、小文件

    # base64编码

    编码的理解：数据从一种形态转换成另一种形态的过程

    base64编码： 将图片二进制数据(不可书写)转换成字符串形式(可书写)

    # 浏览器兼容性

    # 问题产生的原因

    • 市场竞争 以前很严重，现在逐渐缓解

    浏览器厂商为了占取市场份额，不断加入自己的新东西吸引用户而官方标准出来的却很慢

    当前官方标准出来的很快了，各个浏览器厂商都遵守。

    但一些历史遗留问题仍然在。

    • 官方标准变化

    比如旧版本浏览器不支持css3的某些属性(因为那个时候官方标准还未完全确定)，现在基本都支持了

    # 厂商前缀

    属性不是标准的，是厂商自己独有的。 比如： Chrome、safari：-webkit前缀 firfox:-moz前缀 IE:-ms前缀 opera:-o前缀

    # 查找css兼容性

    # 居中总结

    居中：盒子在其包含块中居中

    1. 行盒（行块盒）水平居中

    直接设置行盒（行块盒）父元素text-align:center

    1. 常规流块盒水平居中

    定宽，设置左右margin为auto

    1. 绝对定位元素的水平居中

    定宽，设置左右的坐标为0（left:0, right:0），将左右margin设置为auto

    1. 单行文本的垂直居中

    设置文本所在元素的行高，为整个区域的高度

    1. 行块盒或块盒内多行文本的垂直居中

    没有完美方案

    设置盒子上下内边距相同，达到类似的效果。

    1. 绝对定位的垂直居中

    定高，设置上下的坐标为0（top:0, bottom:0），将上下margin设置为auto

    # 样式补充

    # display:list-item

    设置为该属性的盒子本质上仍然是个块盒子，但同时该盒子会附带另一个盒子!

    元素本身的盒子叫主盒子，附带的盒子叫次盒子。

    次盒子和主盒子水平排列

    1. list-style-type 设置次盒子中内容的类型
    2. list-style-position 设置次盒子相对于主盒子的位置
    3. list-style: none 清空次盒子

    # 图片失效时宽高失效的问题

    如果img元素的链接失效,img元素的特性和普通行盒一样，无法设置宽高

    解决方法： 设置display:inline-block或display:block

    # 行盒中包含行块盒或可替换元素

    现象：行盒的高度与内部高度没关系(因为行盒的高度与字体大小有关)

    解决： 内部：display:block

    # text-align:justify

    分散对齐(最后一行除外)

    如果最后一行也分散对齐，可以给父容器加个::after伪元素并设置display:inline-block;

    # 制作一个三角形

    宽高设置为0，设置边框宽度以及单独设置某一边框颜色

    # direction和writing-mode属性

    开始和结束是相对的，不同国家有不同习惯 左右是绝对的

    /*right to left*/
direction: rtl; /* 文字从右向左 */

/*设置文字书写方向 例子：古文*/
write-mode: vertical-rl; /* 垂直方向上从右向左书写 */
