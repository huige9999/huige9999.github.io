    # 不确定的动态依赖

    # 存在的问题

    const module = document.getElementById("txt").value;
const a = require("./utils/" + module); // 动态依赖
    1
    2

    像这种需要靠用户输入之后才能确定的依赖，webpack 的处理方式是：将 utils 文件夹下的所有文件都打包进来

    # 解决方案

    确定一个范围，让 webpack 只打包这个范围内的文件。

    其实require("./utils/" + module)会被转换成require.context('./utils')(仅在 webpack 运行过程中有效)

    # require.context

    • 参数 1：目录，哪个目录中的模块需要添加到打包结果
    • 参数 2：是否递归寻找子目录，如果为 true，表示需要寻找子目录
    • 参数 3：正则表达式，凡是匹配的才会加入到打包结果
    // 生成一个上下文
const context = require.context("./utils", true, /\.js$/);
// 在这个上下文中查看a模块
const a = context('./a.js')
// 返回上下文中的所有模块
context.keys();

    # 搭建 Node 应用

    # Node开发流程

    流程1：直接开发，直接部署

    1. 搭建 node 工程，直接开发
    2. 开发过程中使用 git 进行管理
    3. 开发完成后，提交 git
    4. 进入部署服务器，从 git 中拉取最新代码，然后npm install (服务器和开发环境的工程结构是一样的)

    存在的问题：

    1. 服务器在npm install的过程中，会占用比较大的网络资源
    2. 代码没有压缩，拉取速度较慢
    3. 开发过程中，无法使用较新的语法
    4. 开发过程中，无法使用 ES6 模块化

    流程2：直接开发，用 webpack 打包，然后部署

    1. 搭建 node + webpack 工程
    2. 开发后，使用 webpack 打包
    3. 将打包结果上传到服务器，服务器直接运行

    生产环境的运行：

    1. 监控源代码目录，如果源代码有变动
    2. 将环境变量设置为development，然后进行打包
    3. 运行打包结果

    # 配置细节

    // 设置跨平台环境变量然后打包
 "build": "cross-env NODE_ENV=production webpack"

// 监控源代码目录，如果代码有变化则执行构建，然后运行dist目录下的代码(保证和生产环境运行的代码一致)
 "dev": "nodemon --watch src --exec 'npm run dev:build && npm run dev:exec'",
 "dev:build": "cross-env NODE_ENV=development webpack",
 "dev:exec": "node dist/index",

    # 配置细节

    webpack.config.js:
// 入口是服务端的Js(客户端打包的只是静态资源是依附于服务端的)
  entry: "./server/index.js",
  target: "node", // 指定打包的目标环境为node 默认为web
  node: false, // 关闭wepback对node的模拟行为


vue.config.js:
module.exports = {
  outputDir: "../dist/public",
};
// 再对客户端进行打包，将打包结果放入dist/public目录下


package.json: // 合并命令 npm-run-all可以运行多个命令 -s 串行执行
 "build": "npm-run-all -s build:server build:client",
 "build:server": "cross-env NODE_ENV=production webpack",
 "build:client": "cd client && npm run build"
