    图的最小生成树问题

    汪元会 9/21/2024 前端算法

    # 1. 什么是图的最小生成树问题？

    拿村庄修路来举例子：希望所有的村庄都联通，所需要的最小花费是多少？

    # 图的最小生成树怎么理解？

    • 树是图的特殊情况(有向无环图)
    • 要想花费最少，就没必要每个村庄都各自直接联通(即：上图可以去掉一些边)

    问题转换为：如何对上图连线进行删减使得最终的花费最少？

    # 2. 为什么要解决图的最小生成树问题？

    上图的节点如果特别多，靠人力就很难算出最小花费，所以需要算法来帮助我们计算。

    # 3. 如何解决图的最小生成树问题?

    • 普利姆算法(加点法)
    • 克鲁斯卡尔算法(加边法)

    # 4. 普利姆算法的思路

    1. 随便选一个点D作为起点，看D的连线哪个最便宜，就把它连上去。

    1. 再把C,D看成一个整体，再看哪个连线最便宜，就把它连上去。

    2. 再把B,C,D看成一个整体，再看哪个连线最便宜，就把它连上去。

    # 4.1 小结感悟

    • 随便选一个点，以该点为起点找最便宜的连线
    • 已连接的多个点当成一个整体重复上面的过程，不过要注意的是：已经联通的点就不要再连了

    # 4.2 算法描述

    1. 任选一个点作为起点
    2. 找到以当前选中点为起点路径最短的边
    3. 如果这个边的另一端没有被联通进来，那么就连起来!
    4. 如果这个边的另一端已经被联通了,则看倒数第二短的边
    5. 重复2-4步，直到所有点都被联通

    # 5.2 代码实现

    /**
 * 普利姆算法的实现
 */

const max = 10000;
const pointSet = [];

// 边集 记录点与点之间的距离
const distance = [
  [0, 9, 7, max, max],
  [9, 0, 8, 6, max],
  [7, 8, 0, 5, max],
  [max, 6, 5, 0, 7],
  [max, max, max, 7, 0],
];

/**
 * 创建图的节点
 */
function Node(value) {
    this.value = value;
    this.neighbors = [];
}

// 创建点集中的节点
const a = new Node("A");
const b = new Node("B");
const c = new Node("C");
const d = new Node("D");
const e = new Node("E");
pointSet.push(a, b, c, d, e);


/**
 * 获取当前这个点在distance表格中的第几行
 */
function getIndex(str) {
  for(let i = 0;i < pointSet.length;i++) {
    if(pointSet[i].value === str) {
        return i
    }
  }
  return -1;
}


/**
 * 根据选中节点获取最小代价的边所连接的节点
 * allPoints 点集
 * distance 边集
 * nowPointSet 当前已经选择的节点集合
 */
function getMinDisNode(pointSet, distance, nowPointSet) {
   let fromNode = null; // 线段的起点
   let toNode = null; // 线段的终点
   let minDis = max; // 最小距离
   // 根据选中的节点为起点，依次判断连接其他的点的距离的值是多少
   for (let i = 0; i < nowPointSet.length; i++) {
     const nowPointIndex = getIndex(nowPointSet[i].value); // 当前点在disance表格中的第几行
      for(let j = 0; j < distance[nowPointIndex].length;j++) { // 遍历distance这一行的所有节点
        const thisNode = pointSet[j];
        if(nowPointSet.indexOf(thisNode) === -1 && distance[nowPointIndex][j] < minDis) {
            // 进入此if说明当前点还没有被联通 并且距离是目前最短的
            fromNode = nowPointSet[i];
            toNode = thisNode;
            minDis = distance[nowPointIndex][j];
        }
      }
   }
   // 最外层循环走完之后，会找到作为起点的节点组的最小距离的边对应的终点节点
   // 连接节点
   fromNode.neighbors.push(toNode);
   toNode.neighbors.push(fromNode);
   return toNode;
}


/**
 * @param {*} pointSet 点集
 * @param {*} distance 边集
 * @param {*} start 起始点
 */
function prim(pointSet, distance, start) {
  const nowPointSet = []; // 存放已经选择的节点
  nowPointSet.push(start);
  // 获取最小代码的边
  while(true) {
    const minDisNode = getMinDisNode(pointSet, distance, nowPointSet);
    // 将获取到的最小距离点加入到节点组中
    nowPointSet.push(minDisNode);
    // 如果节点组的长度等于点集的长度，说明所有节点都已经被联通
    if(nowPointSet.length === pointSet.length) {
        break;
    }
  }
}


prim(pointSet, distance, pointSet[2]);

    # 5.3 代码小结

    主干函数:

    参数：图和起始点

    逻辑简化:

    1. 定义起始点组
    2. 不断地找最小代价点加入起始点组，直到所有点都被联通(即：起始点组的长度等于点集的长度)

    难点：如何找到最小代价的点

    参数：图、起始点组

    逻辑简化：

    • 被比较的对象：起始点组中的每一个点

    • 比较的对象：对应图中的点

    • 定义起始节点、终点节点、最小距离(用于记录每次比较的最优结果)

    • 双层for循环：遍历比较【起点组的每个点】和【对应图中的每个点】 如果有最优解则更新最小距离、起始节点、终点节点

    • 最优解：两点的距离更新 && 【比较对象的点】不在【被比较对象所在的点集】中
