    Vue3 ref和reactive的区别

    汪元会 7/5/2024 Vue3

    # 浅层分析

    # a) 调用方式

    这俩都是函数 接收一个参数 返回一个响应式对象

    const countRef = ref(0)
const countReactive = reactive({ count: 0 })
    1
    2

    # b) 传参类型有点区别

    ref 的参数可以是基本数据类型，也可以是对象，reactive 的参数只能是对象

    const countRef1 = ref(0)
const countRef2 = ref({count: 0})

const countReactive = reactive({ count: 0 })

    # 源码分析

    # a) ref 函数的参数是基本数据类型的情况

    比如：

    const countRef = ref(0)
    1

    它会创建并返回这样一个对象：

    我们可以看到 value 属性其实是个访问器属性，真实的值保存在_value 中。

    • 创建对象的时候会将原始数据保存到_value 中
    • ref 对象.value 会执行 get value()函数，进行数据<->函数绑定
    • ref 对象.value = xxx，会执行 set value()函数,执行绑定的函数(当然这里是异步的 不会立即执行而是加入微队列)

    # b) ref 函数的参数是对象类型的情况

    比如：

       const countRef = ref({
            age: 25,
            name: "John",
          });
    1
    2
    3
    4

    创建对象的时候会比上面多了一步：会将普通对象转换成 reactiveObject(reactiveObject 其实就是一个代理对象)

    这样做的好处是：

    • 无论你是 ref 对象.value = 新的值(拦截.value 的相关操作)
    • 还是 ref 对象.value.代理对象.属性 = 新的值(代理对象能够拦截对目标对象属性的一切操作) 都能触发响应式

    # c) ref 函数的参数也是个 ref 对象的情况

    比如：

      const countRef = ref({
            age: 25,
            name: "John",
          })
          const countRef2 = ref(countRef);
    1
    2
    3
    4
    5

    底层会判断如果参数是个 ref 对象，则直接返回这个 ref 对象，也就是说 countRef2 其实是 countRef 的一个引用。

    # d) ref 函数的参数是 reactiveObject 的情况

    比如：

       const countReactive = reactive({
            age: 25,
            name: "John",
          })
          const countRef = ref(countReactive);
    1
    2
    3
    4
    5

    前面说过当参数是引用数据类型的时候会调用toReactiveObject 函数，将其转化为reactiveObject. 由于参数本身就是 reactiveObject，在调用toReactiveObject时候 会直接将参数返回

    # e) reactive函数

    当ref函数的参数是引用类型的时候其实已经包含了reactive函数的执行情况，这里就不作赘述了。

    # 总结

    用集合表示下ref和reactive功能上的关系：

    • ref的出现是为了解决基本数据类型转成响应式的问题。
    • 如果参数是个引用类型，用ref还是reactive看个人习惯或者规范要求。
